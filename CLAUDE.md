# 出走前チェックリスト - 編集ルール

サーキット走行の出走前チェックリストPWA。iPhoneのホーム画面アプリとして使用。
構成: index.html(全ロジック内包) / sw.js(オフラインキャッシュ) / manifest.json / アイコン。

## 文言の絶対ルール(ユーザー指示・違反禁止)

1. **文言を勝手に短縮しない。** 「ヘルメット・HANSクリップ装着」のような
   圧縮表記は禁止。各動作の動詞(かぶる・締める・装着・押す・開ける等)を
   必ず残す。複数動作を1ステップに統合する場合は `\n` で行を分け、
   各行に動詞を含める。
2. **確認項目(〜か確認)を sub(小さい補足文字)に格下げしない。**
   チェック項目は大きい本文(text)に置く。sub は理由や補助情報のみ。
3. **ステップの統合・削除・並べ替え等の構成変更は、必ず事前にユーザーへ
   確認してから行う。** 文言の意味を変える変更も同様。

## 動作フローの前提(変更時は要注意)

- グローブ装着後はタッチ操作不可。**タップが必要な操作はすべて
  グローブ装着前に完了**する構成にすること。
- RaceChrono の起動は `shortcuts://run-shortcut?name=RaceChrono`
  (iOSショートカット経由)。`racechrono://` 直接はインポート画面に
  なるため使用禁止。
- 各ページはチェックボックス式: 項目ボタンをタップで✓(再タップで解除)。
  **全項目チェックで次ページへ自動遷移**。画面タップで進む方式は廃止済み。
- 最終画面(赤)は全項目チェックで openRaceChrono() を呼び、
  「素手でチェック→RaceChrono起動→戻らずにそのまま走行」の流れ。
- チェック状態は localStorage(checklist_checked)に保存。全✓状態は
  復元しない(安全側に倒して再確認させる)。

## 技術ルール

- **index.html か sw.js を変更したら、必ず sw.js の CACHE バージョンを
  1つ上げる**(例: checklist-v21 → checklist-v22)。
- sw.js はページ本体(navigate)をネットワーク優先(2.5秒タイムアウト、
  圏外時はキャッシュ)、その他はキャッシュ優先。この方式を維持する。
- デプロイは GitHub Pages(main ブランチ)。ユーザーの指示により
  main へ直接プッシュしてよい。
- 変更後はヘッドレスChromiumでレンダリング・遷移のスモークテストを行う。
